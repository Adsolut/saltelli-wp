# v0.27.2 Attorney Container Alignment Fix

**Date:** 2026-05-02
**Version:** `0.27.2-beta-attorney-container-align`
**Commit:** `53ccfd5`

## Issue (segnalata Duccio)

> "Sotto biografia gli altri paragrafi partono come indentati in un'altra tab"

**Visualmente:**
- Bio drop-cap + paragraph: aligned a `x=408` (col 2 edge dell'article-grid) ✓
- Sei aree / Formazione / Casi / Articoli / CTA: aligned a `x=480` (72px shift right) ❌

User percepiva uno "step indent" sotto biografia, layout incoerente.

## Root cause

Article-grid v0.27.0 applica `padding-inline: clamp(24, 5vw, 96)` al wrapper article. Al viewport 1440 = 72px padding.

Ogni `.sl-attorney__{section}` markup wrappa contenuto in `<div class="sl-container">` che AGGIUNGE altri 72px di padding-inline (default `--container-pad: clamp(24px, 5vw, 96px)`).

v0.27.0 aveva override SOLO bio:
```css
.sl-attorney__bio .sl-container { padding-inline: 0; }
```

Ma `competenze`, `aree`, `timeline`, `casi`, `articles`, `cta` mantenevano default → 72px shift orizzontale.

**JSX architecture verificata** (`saltelli-s2-attorney-single.jsx:106-244`):

```jsx
<section grid 240px/1fr>
  <aside sticky />
  <div>             ← UN SOLO container per tutti body content
    <div>Bio</div>
    <div>Sei aree</div>
    <div>Formazione</div>
    <div>Casi</div>
    <div>CTA</div>
  </div>
</section>
```

Tutti body content come `<div>` siblings dentro singolo wrapper. Zero nested container. Edge sinistro uniforme.

Nostro PHP usa `<section>` separati, ognuno con `<div class="sl-container">`. Per replicare la stessa edge alignment senza markup change → reset container padding-inline.

## Fix v0.27.2

```css
@media (min-width: 1024px) {
    .sl-attorney > .sl-attorney__bio .sl-container,
    .sl-attorney > .sl-attorney__competenze .sl-container,
    .sl-attorney > .sl-attorney__aree .sl-container,
    .sl-attorney > .sl-attorney__timeline .sl-container,
    .sl-attorney > .sl-attorney__casi .sl-container,
    .sl-attorney > .sl-attorney__articles .sl-container,
    .sl-attorney > .sl-attorney__cta .sl-container {
        padding-inline: 0 !important;
        max-width: none !important;
    }
}
```

Mobile (`< 1024`): bio container mantiene padding-inline ridotto per leggibilità (resto delle sezioni eredita default container per stack 1-col).
