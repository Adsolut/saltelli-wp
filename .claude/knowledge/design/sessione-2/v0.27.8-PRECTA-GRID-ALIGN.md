# v0.27.8 Pre-CTA Grid Alignment Cross-Page

**Date:** 2026-05-02
**Version:** `0.27.8-beta-precta-grid-align`
**Commit:** `2a245df`

## Issue (segnalata Duccio)

> "Ne abbiamo un altro cross pages che è il footer con un allineamento orizzontale diverso rispetto a quello nelle pagine mi riferisco a § Ultima chiamata"

## Diagnosi

Il footer pre-CTA ("§ Ultima chiamata · Vorresti raccontarci la tua pratica?") usava grid `8fr/4fr` con:
- Col 1 (8fr ~800px): eyebrow + h2 + p TOGETHER
- Col 2 (4fr ~400px): button + trust

Risultato: eyebrow + h2 entrambi a `x=72` (allineati a sinistra).

Le sezioni page (chi-siamo founding/team/principles/timeline/CTA) usano grid `3fr/9fr`:
- Col 1 (3fr ~300px): eyebrow ONLY
- Col 2 (9fr ~924px): h2 + content

Con eyebrow a `x=72` e h2 a `x=444`.

**Visualmente:** scrollando dalla last page section (h2 a x=444) al footer pre-CTA (h2 a x=72), l'h2 saltava 372px a sinistra → break del rhythm visuale verticale.

## Audit pre-fix

```
chi-siamo CTA "Prenoti una consulenza gratuita":
  eyebrow x=72  · h2 x=444   (3fr/9fr)

pre-CTA "Vorresti raccontarci la tua pratica?":
  eyebrow x=72  · h2 x=72    (8fr/4fr)  ← MISALIGN

Δ h2 = -372px (footer saltava sinistra)
```

## Fix v0.27.8 — CSS-only via display:contents

```css
@media (min-width: 1024px) {
    .sl-foot-cta {
        grid-template-columns: 3fr 9fr;   /* era 8fr 4fr */
        gap: 64px;                         /* era 96px */
        align-items: start;                /* era end */
    }
    /* Lede wrapper trasparente: figli partecipano direttamente al grid */
    .sl-foot-precta__lede {
        display: contents;
    }
    .sl-foot-precta__eyebrow {
        grid-column: 1;
        grid-row: 1;
    }
    .sl-foot-precta__h2     { grid-column: 2; grid-row: 1; }
    .sl-foot-precta__p      { grid-column: 2; grid-row: 2; }
    .sl-foot-precta__action {
        grid-column: 2;
        grid-row: 3;
        margin-top: 32px;
    }
}
```

**Trick:** `display: contents` su `.sl-foot-precta__lede` "dissolve" il wrapper, permettendo a eyebrow + h2 + p di partecipare DIRETTAMENTE come grid items di `.sl-foot-cta`. Niente markup change.

## JSX vs Override decisione

JSX source-code (saltelli-s2-footer.jsx line 160-164) usa `8fr 4fr` con button in separate col. **Override visual-design intent (Duccio review):** preferire pattern 3fr/9fr per consistency cross-page con altre section CTA.

Trade-off: button non più in col 4fr separato (a destra) ma stacked sotto p (in col 9fr). Più coerente verticalmente con rhythm sezioni page.
