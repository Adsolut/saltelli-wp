## Issue critici trovati & risolti durante walkthrough

### 🐛 ISSUE 1 — Article-grid row collision (attorney)
**Sintomo:** sticky aside (TEL/EMAIL/WhatsApp) renderizzato sopra il breadcrumb invece che sotto l'hero. Hero in row 2 invece di row 1.

**Diagnosi:** v0.27.0 CSS aveva `aside { grid-row: 2 / -1 }` ma header/hero senza `grid-row` esplicito. Auto-flow grid + grid-row span misto causava aside a finire in row 1 (con hero spinto in row 2).

```
articleGridRows pre-fix: 211.594px 721.781px ...   ← row 1 = aside! row 2 = hero!
asideRect pre-fix:        y=120 (sopra hero)
```

**Fix v0.27.1:**
- `header { grid-row: 1 }` esplicito (anchor)
- `aside { grid-row: 2 / span 100 }` deterministico

```
articleGridRows post-fix: 857.781px 528.484px 981.391px ...   ← row 1 = HERO!
asideRect post-fix:        y=942 (dopo hero) ✓
```

### 🐛 ISSUE 2 — Portrait max-width 480 inside hero col 616
**Sintomo:** portrait (foto Emiliano) renderizzata a 480×480 invece di 616×616 (riempire intera col 1 dell'hero-grid 1fr/1fr).

**Diagnosi:** FIX F3 v0.8.1 aveva `figure.sl-attorney__portrait { max-width: 480px !important; aspect-ratio: 3/4 }` per evitare overflow su layout precedente. Con la nuova hero-grid 1fr/1fr (cella 616px), il limite 480 lasciava 136px gap.

**Fix v0.27.1:** override scoped solo dentro `.sl-attorney__hero-grid` context:
```css
.sl-attorney__hero-grid figure.sl-attorney__portrait {
    max-width: none !important;
    width: 100% !important;
}
```

Aspect-ratio già override 1/1 desktop in v0.27.0.

```
portraitWidth post-fix: 616px (riempie col)
portraitMaxWidth post-fix: none
```
