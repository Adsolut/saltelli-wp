## 🔧 Refactor patterns applicati

### 1. ACF reads con fallback grazioso

Pattern: `saltelli_field('field_name', $pid, 'hardcoded fallback')`
- Se ACF popolato (Wave 2): ritorna valore ACF.
- Se ACF empty: ritorna fallback hardcoded.
- Frontend invariato in entrambi i casi.

### 2. CPT query replace hardcoded array

Pre-Wave 3 (page.php):
```php
$sl_faq_topics = [...]; // hardcoded 28 items in 6 topics
```

Post-Wave 3 (template-parts/page-faq.php):
```php
foreach ($topic_meta as $topic_slug => $meta) {
    $faqs = get_posts(['post_type' => 'saltelli_faq', 'tax_query' => [...]]);
    // Iterate via ACF field reads
}
```

CPT replaced arrays in:
- `/faq/` → 28 items via `saltelli_faq` + `faq_topic` taxonomy
- `/casi/` → 10 items via `saltelli_caso` + `caso_categoria` taxonomy
- `/costi/` § 01 → 3 modalità via `saltelli_modalita`
- `/costi/` § 02 → 3 scenari via `saltelli_scenario`
- `/costi/` § 05 → 4 trust plates via `saltelli_trust`
- `/chi-siamo/` § 04 → 3 principi via `saltelli_principio`

### 3. Field rename con back-compat

Single-templates leggono nuova schema → fallback legacy:
```php
// Wave 3: prefer hero_role (Wave 1) → fallback ruolo_breve (Wave 0).
$ruolo = (string) saltelli_field('hero_role', $pid, '');
if ($ruolo === '') $ruolo = (string) saltelli_field('ruolo_breve', $pid, '');
```

Renames applicati:
- `ruolo_breve` (Wave 0) → `hero_role` (Wave 1) [single-avvocato.php]
- `is_tier_1_focus` (Wave 0) → `is_tier_1` (Wave 1) [single-competenza.php]

Tutti gli altri field name match già 1:1.

### 4. Theme Options binding

footer.php + header.php read Wave 1 schema con fallback Wave 0:
```php
$tel = saltelli_option('studio_telefono_pubblico', '');
if ($tel === '') $tel = saltelli_option('contact_telefono_pubblico', '+39 081 1813 1119');
```

Bound fields:
- studio_indirizzo_via + studio_cap_citta + studio_quartiere → costruisce `$ftr_indirizzo`
- studio_telefono_pubblico → `$ftr_tel` + header click-to-call
- studio_email → `$ftr_email`
- studio_pec → `$ftr_pec`
- studio_piva → `$ftr_piva`
- saltelli_studio_data() resta come ultimate fallback
